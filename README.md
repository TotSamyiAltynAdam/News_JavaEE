**SET UP**
---

1. Прежде чем начать проект, мы с вами попробуем установить сервер - Tomcat. 
Tomcat нам нужен для запуска веб сервера на Java. Tomcat - это программное обеспечение с открытым исходным кодом, 
разработанное фондом Apache. Веб-страницы сами по себе являются статическими файлами HTML. Следовательно, 
клиент не может взаимодействовать со статической веб-страницей. Чтобы наши веб-страницы имели динамические возможности,
требуется веб-сервер.

  
2. Для того чтобы скачать этот сервер, нужно будет зайти на официальный сайт -  http://tomcat.apache.org/
В данном сайте выберите версию: Если у вас версия Java 8, то используйте Tomcat 8, если выше то Tomcat 9 и.т.д.


3. Есть такой сайт - https://mvnrepository.com/
С этого сайта можно скачать любую библиотеку для Java. В нашем случае, в поисковик:
   - Мы введем: javax servlet. Выбираем первый результат - Java Servlet API
   - Далее, точно также, в поисковик вводим слово: javax.servlet.jsp.
   Выбираем первый вариант - JavaServer Pages(TM) API.


4. Далее, копируем и вставляем наши jar файлы в эту папку lib


5. Затем, мы добавим эти библиотеки в наш проект через настройки структуры проекта: 
File => Project Structure => Libraries


6. Далее, сразу надо будет создать веб модуль. Для этого создадим специальный файл - web.xml
(Deployment Descriptor / Дескриптор развертывания), специальный конфигурационный файл артефакта, 
который будет развернут в контейнере сервлетов. Дескриптор развертывания описывает то, как компонент, 
модуль или приложение должно быть развернуто. Для веб-приложений дескриптор развертывания должен называться web.xml 
и находится в папке WEB-INF, в корне веб-приложения.


7. Далее, нужно будет настроить веб модуль нашего проекта. 
Заходим в File => Project Structure => Modules, нажимаем на "+", и внизу выбираем "Web"


8. Если вы хотите вручную создать артефакт, то также заходите в File => Project Structure => Artifacts. 
Затем нажимаете на "+" и выбираете "Web Application: Exploded" затем "From Modules" и выбираем наш проект.
Артефакт нам нужен для того чтобы завернуть весь наш проект в единый файл или папку. 
В нашем случае в папку - war (Web Application Resource).


9. Ну и в конце, мы должны будем подключить наш сервер Tomcat и настроить конфигурацию.
Нажимаем на кнопку "Add Configuration" в верхнем меню.
Затем добавляем конфигурацию сервера, нажимая на кнопку "+" выбираем "Tomcat Server" и далее "Local".

Java Server Pages
---

JSP (JavaServer Pages) - технология, позволяющая веб-разработчикам создавать содержимое веб страницы, 
которое имеет как статические, так и динамические компоненты. 
Страница JSP содержит текст двух типов: 
   - статические исходные данные, которые могут быть оформлены в одном из текстовых форматов HTML, SVG, WML, или XML, 
   - и JSP- элементы, которые конструируют динамическое содержимое.

---
Java DataBase Connectivity
---

JDBC (Java DataBase Connectivity) - платформенно-независимый промышленный стандарт взаимодействия 
Java-приложений с различными СУБД, реализованный в виде пакета java.sql, входящего в состав Java SE. 
JDBC основан на концепции так называемых драйверов, позволяющих получать соединение с базой данных 
по специально описанному URL. Драйверы могут загружаться динамически (во время работы программы).


1. В этом проекте мы будем использовать MySQL. Для этого нам нужно будет скачать драйвер, 
с официального сайта Maven Repository по ссылке:
https://repo1.maven.org/maven2/mysql/mysql-connector-java/8.0.15/mysql-connector-java-8.0.15.jar
После загрузки, нужно будет добавить данный JAR файл в наш проект, как мы добавляли библиотеки сервлета и JSP.

2. Далее, нам нужно будет установить MYSQL сервер. Для этого достаточно скачать MAMP с официального сайта:
https://www.mamp.info/en/downloads/
